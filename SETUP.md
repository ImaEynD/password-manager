# Инструкция по локальному запуску

Данное руководство поможет вам развернуть проект локально за несколько минут. 

> **Важно:** Файлы зависимостей (`node_modules/`, `target/`) и конфигурации окружения (`.env`) намеренно исключены из репозитория через `.gitignore` в целях безопасности и чистоты кода.

---

## 1. Предварительные требования

Перед началом убедитесь, что в вашей системе установлены:
- **Java 21** (или выше)
- **Node.js** (версия 20.x LTS или выше)
- **PostgreSQL** (версия 15 или 16)
- **Maven** (или используйте встроенный в вашу IDE, например, IntelliJ IDEA)

---

## 2. Настройка базы данных

Создайте пустую базу данных для проекта. 

> **Примечание для пользователей WSL2 / Ubuntu:**  
> В некоторых конфигурациях PostgreSQL по умолчанию слушает порт **5433**, а не стандартный 5432. Если у вас стандартный порт, просто замените `5433` на `5432` в следующих шагах.

```
# Создаем базу данных (выполняется от имени пользователя postgres)
sudo -u postgres createdb password_manager
```

---
## 3. Конфигурация окружения 
Приложение реализует паттерн Externalized Configuration (12-Factor App). Вы можете переопределить настройки по умолчанию, создав файл .env.
1. Перейдите в директорию `backend/`.
2. Создайте файл с именем `.env` (он уже добавлен в `.gitignore` и не попадет в Git).
3. Добавьте в него следующие строки, заменив значения на свои:
```
# Настройки подключения к PostgreSQL
DB_URL=jdbc:postgresql://127.0.0.1:5433/password_manager
DB_USER=postgres
DB_PASSWORD=ваш_пароль_от_postgresql

# Секретный ключ для генерации JWT (минимум 32 символа)
JWT_SECRET=my-super-secret-local-jwt-key-for-development-only

```

Если файл `.env` не создан, Spring Boot автоматически использует безопасные значения по умолчанию, указанные в `src/main/resources/application.properties`.

---

## 4. Запуск Backend
Находясь в корневой папке проекта, перейдите в `backend/` и выполните:
```
cd backend
# 1. Очистка и загрузка зависимостей (сборка)
mvn clean install

# 2. Запуск Spring Boot приложения
mvn spring-boot:run
```

Успех: В консоли вы увидите сообщение Started PasswordManagerApplication in `X.XXX` seconds. API доступен по адресу: `http://localhost:8080`

---

## 5. Запуск Frontend
Откройте новый терминал (не закрывая `backend`), перейдите в папку `frontend/` и выполните:
```
cd frontend

# 1. Установка зависимостей (обязательно, так как папка игнорируется Git)
npm install

# 2. Запуск сервера разработки Vite
npm run dev
```
Успех: В консоли появится ссылка (обычно `http://localhost:5173`). Откройте её в браузере.

## Решение частых проблем
| Проблема | Возможная причина и решение |
| :--- | :--- |
| `Connection refused` при запуске `Backend` | `PostgreSQL` не запущен или используется неверный порт. Проверьте статус службы (`sudo service postgresql status`) и убедитесь, что в `.env` или `application.properties` указан правильный порт (`5432` или `5433`) и хост `127.0.0.1` (вместо `localhost` для избежания проблем с `IPv6` в `WSL`). |
| `AEADBadTagException` при попытке войти |Рассинхронизация криптографической сессии. Убедитесь, что вы используете актуальную версию кода, где фронтенд вызывает эндпоинт `/api/passwords/unlock` сразу после успешного логина. |
| `Cannot find module 'vite'` | Вы забыли выполнить `npm install` в папке `frontend/` перед запуском `npm run dev`. |
| Ошибки прав доступа в WSL | Если файлы не сохраняются, выполните `sudo chown -R $USER:$USER .` в корне проекта, чтобы вернуть себе права владельца. |

## Проверка работоспособности
1. Откройте `http://localhost:5173`
2. Нажмите "Зарегистрироваться".
3. Создайте аккаунт (например, логин: `admin`, пароль: `Admin123456!`).
После автоматического входа добавьте тестовую запись (например, сервис: GitHub).
4. Обновите страницу. Запись должна корректно расшифровываться и отображаться.
5. Если все шаги пройдены успешно — поздравляем, проект полностью развернут!
